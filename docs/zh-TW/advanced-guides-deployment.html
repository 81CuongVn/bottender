Deployment

一旦你在開發環境完成了你的機器人，下一步就是部署你的機器人到託管服務上。

在開始之前

我們假設你已經在開發環境建立好至少一個基本的 Bottender 應用程式。 如果你還沒有，你可以查看 Getting Started 在幾分鐘內建立你的第一個 Bottender 應用程式，然後跳到你最愛的聊天平台的設定文件：

  • Messenger Setup
  • WhatsApp Setup
  • LINE Setup
  • Slack Setup
  • Telegram Setup
  • Viber Setup

Heroku

Heroku 是最知名的託管服務之一。 不只是乾淨的文件，易於擴充，使用 Git 進行部署，而且有友善的免費方案給實驗目的。

接下來，你可以看到 Heroku 部署的必要步驟：

第一步：註冊 Heroku 帳號以及下載 Heroku CLI

首先，如果你還沒有 Heroku 帳號就註冊一個，然後下載和安裝 Heroku CLI。

注意： 為了完整的指令列表，請參閱 Heroku 的文件，Heroku CLI 指令。

第二步：Heroku 登入和建立 Heroku 應用程式

在開始之前，確保你有登入 Heroku 帳號透過：

heroku login

然後，你可以透過以下指令建立 Heroku 應用程式。

heroku create <your-heroku-app-name>

注意：如果你沒遵守，你可能會看到一些應用程式名稱規則 例如：名稱必須以字母開始，以字母或數字結束並且只能包含小寫字母、數字以及破折號

一旦你成功建立你的 Heroku 應用程式，你會看見一個像 https://<your-heroku-app-name>.herokuapp.com/ 的應用程式部署地址。 為了之後的 Webhook 設定，你可以把它記下來。

第三步：填寫 Heroku 上的環境變數

使用以下指令設置 Heroku 應用程式的環境變數：heroku config:set -a <your-heroku-app-name> <ENV_KEY_01>=<ENV_VALUE_01>.

當聊天平台需要多個環境變數時，你可以使用像 heroku config:set -a <your-heroku-app-name> <ENV_KEY_01>=<ENV_VALUE_01> <ENV_KEY_02>=<ENV_VALUE_02> 的指令。

例如：

heroku config:set -a <your-heroku-app-name> MESSENGER_PAGE_ID=xxxxxx MESSENGER_ACCESS_TOKEN=xxxxxx MESSENGER_APP_ID=xxxxxx MESSENGER_APP_SECRET=xxxxxx MESSENGER_VERIFY_TOKEN=xxxxxx

第四步：在你的 Bottender 應用程式中使用 Git

Heroku 的部署依賴 Git。 確保你有在你的 Bottender 應用程式當中執行過 git init 以及做出第一個 Commit。

例如：

git init
git add .
git commit -am "first commit"

第五步：部署你的機器人到 Heroku 上並設定 Webhook

當你嘗試設定 Webhook 時，有些聊天平台（例如 Messenger）可能會要求立即進行機器人 Server 驗證。 所以，我們建議你在 Bottender 應用程式 Server 跑起來之後再設定 Webhook。

這裡有兩種設定 Webhook 的方式。

  1. 透過 Developer Console UI 來設定 Webhook，例如：Messager、LINE、Slack
  2. 透過 CLI 來設定 Webhook，例如：Messager、Telegram、Viber

第五之一步：透過 Developer Console UI 來設定 Webhook

透過 Git Push 使用 Heroku CLI 來完成部署。

heroku git:remote -a <your-heroku-app-name>
git push heroku master

然後填入你的 Webhook URL 到聊天平台的 Developer Console。

注意：如果你對於設定 Webhook 設定不熟，你可以參考 Bottender 的文件，Messenger Setup、LINE Setup 以及 Slack Setup。

注意： 如果你沒有修改過你 bottender.config.js 中的 Webhook 路徑，在預設情況下，你的 Messenger 機器人 Webhook 路徑是 https://<your-heroku-app-name>.herokuapp.com/webhooks/messenger；你的 LINE 機器人 Webhook 是 https://<your-heroku-app-name>.herokuapp.com/webhooks/line 等等。

第五之二步：透過 CLI 來設定 Webhook

你可以從 Heroku 的 Procfile 特色獲益，它可以指定應用程式的啟動指令。 我們將使用兩種 Procfile 的程序類型：

  • web 程序類型：告訴 Heroku 在每一個 dyno 中運行你的機器人 Server。
  • release 程序類型：在新版本部署到正式環境之前設定 Webhook。

注意： 有關 Procfile 的更多資訊，請參閱 The Procfile。

以 Messenger 機器人為範例，採用預設的 Webhook 路徑設定時，Procfile 看起來如下：

// Procfile

web: npm start
release: echo "Y" | npx bottender messenger webhook set -w https://<your-heroku-app-name>.com/webhooks/messenger

注意：

  • 這裡的 echo "Y" 是為了回答第一個互動式 CLI 提示。
  • 如果你沒有修改過你 bottender.config.js 中的 webhook 路徑，在預設情況下，你的 Messenger 機器人 webhook 路徑是 https://<your-heroku-app-name>.herokuapp.com/webhooks/messenger；你的 LINE 機器人 webhook 是 https://<your-heroku-app-name>.herokuapp.com/webhooks/line 等等。

最後，你可以透過 Git Push 來使用 Heroku CLI 完成部署並且讓 Heroku 運行 Procfile 來幫助你完成 Webhook 的設定。

heroku git:remote -a <your-heroku-app-name>
git push heroku master

第六步：完成！

恭喜！ 你已經使你的 Bottender 機器人正式環境準備就緒。 分享你超棒的機器人給你的朋友們！

注意： 如果你想要分享你的 Bottender 機器人到社群，請隨時加入你的專案到 Bottender Users。

ZEIT Now 2.0

ZEIT Now 2.0 是一個對開發者超級友善的託管服務。 在部署靜態虛擬主機時，ZEIT Now 以一行魔術指令 now 聞名。

然而，這需要一些設置才能部署 Bottender 應用程式並且使它在 Serverless 的情況下工作。 接下來，你可以看到 ZEIT Now 2.0 部署的必要步驟：

第一步：註冊 ZEIT 帳號

首先，如果你還沒有 ZEIT 帳號的話，註冊一個。

第二步：安裝 ZEIT Now CLI

我們喜愛使用 CLI 來部署！ 透過 npm 來安裝 ZEIT Now CLI：

npm install -g now

第三步：建立 server.js

ZEIT Now 2.0 不支援 npm 腳本，所以我們需要 server.js 作為進入點。

// server.js

const bodyParser = require('body-parser');
const express = require('express');
const { bottender } = require('bottender');

const app = bottender({
  dev: process.env.NODE_ENV !== 'production',
});

const port = Number(process.env.PORT) || 5000;

const handle = app.getRequestHandler();

app.prepare().then(() => {
  const server = express();

  server.use(
    bodyParser.json({
      verify: (req, _, buf) => {
        req.rawBody = buf.toString();
      },
    })
  );

  server.all('*', (req, res) => {
    return handle(req, res);
  });

  server.listen(port, (err) => {
    if (err) throw err;
    console.log(`> Ready on http://localhost:${port}`);
  });
});

第四步：更新你的 now.json

在接下來的配置中，你將進行一些設定：

  • 將 ZEIT Now 設定為 2.0 版
  • 使用 @now/node 來綁定 server.js
  • 將 bottender.config.js 和 index.js 放入 includeFiles。 ZEIT Now 會在背後使用 ncc 和 webpack bundler，所以你需要告訴它們將這兩個文件放入 Bundle
  • 路由所有 (/.*) 到 server.js
  • 加入你的聊天平台特有的環境變數從 .env 到 env。 環境變數的數量會隨著你使用的聊天平台而有所不同。
  • (可選) 如果你在對你的應用程式進行偵錯，你可以設定 DEBUG 環境變數為 bottender*,messaging-api*

使用 Messenger 機器人作為範例。 你的 now.json 是像：

// now.json

{
  "version": 2,
  "builds": [
    {
      "src": "server.js",
      "use": "@now/node",
      "config": {
        "includeFiles": [
          "bottender.config.js",
          "index.js"
        ],
        "bundle": true
      }
    }
  ],
  "routes": [
    {
      "src": "/.*",
      "dest": "/server.js"
    }
  ],
  "env": {
    "MESSENGER_PAGE_ID": "xxxxxx",
    "MESSENGER_ACCESS_TOKEN": "xxxxxx",
    "MESSENGER_APP_ID": "xxxxxx",
    "MESSENGER_APP_SECRET": "xxxxxx",
    "MESSENGER_VERIFY_TOKEN": "xxxxxx",
    "DEBUG": "bottender*,messaging-api*"
  }
}

第五步：更新你的 bottender.config.js

ZEIT Now 2.0 是 Serverless 的託管服務，而且 Serverless Function 會在 HTTP 回覆後就終止。 所以你需要使 Bottender 以同步方式工作，例如，在接收到事件的當下就回覆到聊天平台。

為了讓 Bottender 運行於同步模式，你必須在 bottender.config.js 配置聊天平台為 sync: true。

// bottender.config.js

module.exports = {
  // ... skip

  channels: {
    messenger: {
      enabled: true,
      sync: true,
      // ...skip
    },
    line: {
      enabled: true,
      sync: true,
      // ...skip
    },
    telegram: {
      enabled: false,
      sync: true,
      // ...skip
    },
    slack: {
      enabled: false,
      sync: true,
      // ...skip
    },
    viber: {
      enabled: false,
      sync: true,
      // ...skip
    },
  },
};

第六步：更新你的 package.json

在 engines 環境變數設定 Node.js 版本為 10.x 或 12.x 來避免不推薦使用 node 8 的警告。

// package.json
{
 // ...skip
 "engines": { "node": "12.x" }
}

第七步：使用 now 來部署

在所有設定都完成後，你已經準備好要使用魔術詞，now，來進行 ZEIT Now 2.0 部署。

now

然後你會看到類似以下截圖的內容。

注意： 你的部署 URL 應該像 https://<your-app-name>.<your-user-name>.now.sh

第八步：Webhook 設定

如果你沒有修改過你 bottender.config.js 中的 Webhook 路徑，在預設情況下，你的 Messenger 機器人 webhook 路徑是 https://<your-heroku-app-name>.herokuapp.com/webhooks/messenger；你的 LINE 機器人 Webhook 是 https://<your-heroku-app-name>.herokuapp.com/webhooks/line 等等。

這裡有兩種設定 webhook 的方式。

  1. 透過 Developer Console UI 來設定 Webhook，例如：Messager、LINE、Slack
  2. 透過 CLI 來設定 Webhook，例如：Messager、Telegram、Viber

第八之一步：透過 Developer Console UI 來設定 Webhook

填入你的 Webhook URL 到聊天平台的 Developer Console。

注意：如果你對於設定 Webhook 設定不熟，你可以參考 Bottender 的文件，Messenger Setup、LINE Setup 以及 Slack Setup。

第八之二步：透過 CLI 來設定 Webhook

確保你有和你本地的 .env 檔以及 now.js 相同的環境變數設定。 然後你可以使用以下指令來設定 Webhook。

以 Messenger 機器人為例，你的指令應該像這樣：

npx bottender messenger webhook set -w https://<your-app-name>.<your-user-name>.now.sh/webhooks/messenger

第九步：完成！

恭喜！ 你已經使你的 Bottender 機器人正式環境準備就緒。 分享你超棒的機器人給你的朋友們！

注意： 如果你想要分享你的 Bottender 機器人到社群，請隨時加入你的專案到 Bottender Users。
