Sending Slack Messages

由於 Slack 是專門設計給團隊溝通用的通訊平台，因此通常期待 Slack 機器人能在頻道內或一對一聊天工作。

在頻道內聊天可以有效的收集個人偏好。 例如，我們有一個 Slack 機器人收集同事下午茶訂單。

另外，Slack 機器人特別擅長通知的情境，例如，監控線上伺服器的健康狀態，或 Bottender GitHub repo 的目前星星數（喜歡的話幫忙按個星星！）。 因為 Slack 應用程式可以免費發送訊息到頻道、群組或用戶。

當使用官方 Slack API 開發時，開發者必須管理每個對話框的 token、頻道 ID、用戶 ID。 Bottender 協助開發者透過 context 管理這些參數。機器人開發者可以讓機器人在當下 context 作回覆，不論這個 context 是在 Public Channels、Private Channels、Private Groups 還是 Direct Messaging。

注意：

  • 一個機器人對應一個 Slack 的應用程式這個概念，當你在對照閱讀 Bottender 以及 Slack 的官方文件時是很重要的。
  • 由於 Slack 是複雜的通訊平台之一，因此在這份文件中，我們只會提到最常用的方法。 想看 Bottender 支援的所有 Slack API 列表的話，你可以參考 Bottender's API。 如果你對 Slack 的完整方法有興趣，你可以參考 Slack 的官方文件，"API Methods"。 別錯過在每個方法旁邊的 Test 分頁；他提供了方便的方式來更好地測試原生 Slack API。

發送文字訊息

文字訊息 是在所有通訊平台中最常見以及最常使用的訊息類型 。 當你想攜帶一些動態資料時，它提供了最小型的格式，例如：股價和天氣資訊。

async function SendHi(context) {
  await context.sendText('Hi!');
}

發布訊息

這個方法發布訊息到公開頻道、私密頻道或直接訊息。 為了完整支援 Slack 的聊天 UI 功能，Bottender 傳遞訊息參數到 Slack 的 API。

await context.chat.postMessage({
  text: 'Hello world!',
});

更多的資訊，請參考 Slack 官方文件 chat.postMessage。

更新訊息

更新現有訊息是 Slack 特有的功能之一。 當你的機器人呈現一個即時日期而且你不想要占版面時，這很有用。

在上圖中，第二則訊息是由 Bottender 更新。 你可能會注意到當一個用戶更新他的訊息時，會在訊息旁邊顯示一個小標記 (edited)，機器人更新訊息時不會有 (edited) 標記。

await context.chat.update({
  text: 'Hello world!',
  ts: '1405894322.002768',
});

更多的資訊，請參考 Slack 官方文件 chat.update。

注意：

  • 上面的程式碼中的 ts 參數是指訊息的時間戳記，它是每個頻道內每則訊息的唯一ID。
  • 透過 chat.postEphemeral 或其他方法建立的 Ephemeral 訊息無法被這個方法更新。

刪除訊息

這個方法刪除頻道中由機器人發布的訊息。

await context.chat.delete({
  ts: '1405894322.002768',
});

更多的資訊，請參考 Slack 官方文件 chat.delete。

注意：上面的程式碼中的 ts 參數是指訊息的時間戳記，它是每個頻道內每則訊息的唯一ID。

發布 Ephemeral 訊息

機器人可以在特定公開頻道、私密頻道或私密對話中發布只顯示給指定用戶的 Ephemeral 訊息。 簡而言之，Ephemeral 訊息 允許機器人在目前 Bottender context 下回覆 context 相關的訊息給用戶，例如，當用戶輸入 check-in，機器人可以透過 Ephemeral 訊息 發布 check-in 時間給用戶。

更多的資訊，請參考 Slack 官方文件 chat.postEphemeral。

注意： Ephemeral 訊息是不保證送達的－用戶必須正在使用 Slack 而且是特定頻道的成員。

await context.chat.postEphemeral({
  text: 'Hello world!',
});

更多的資訊，請參考 Slack 官方文件 chat.postEphemeral。

Sharing Me Messages

分享 me 訊息到頻道。

await context.chat.meMessage({
  text: 'Hello world!',
});

更多的資訊，請參考 Slack 官方文件 chat.meMessage。

取得特定訊息的永久連結

取得特定訊息的永久連結

await context.chat.getPermalink({
  messageTs: '1234567890.123456',
});

更多的資訊，請參考 Slack 官方文件 chat.getPermalink。

注意：上面的程式碼中的 messageTs 參數是指訊息的時間戳記，它是每個頻道內每則訊息的唯一ID。

排程訊息

Slack 有內建的訊息排程系統。 其他的通訊平台，為了排程訊息你可能必須跑一個 cronjob。

排程訊息

將訊息排程到一個頻道。

await context.chat.scheduleMessage({
  text: 'Hello world!',
  postAt: '299876400',
});

更多的資訊，請參考 Slack 官方文件 chat.scheduleMessage。

注意： 上面程式碼中的 postAt 參數格式是 Unix EPOCH 時間戳記。

取得排程訊息列表

你可以透過以下程式碼取得排程訊息列表：

await context.chat.scheduledMessages.list();

更多的資訊，請參考 Slack 官方文件 chat.scheduledMessages.list。

刪除已排程的訊息

首先，你必須用 chat.scheduledMessages.list 取得你想刪除的 <SCHEDULED_MESSAGE_ID>。 然後你可以用以下方法來刪除一個已排程的訊息：

await context.chat.deleteScheduledMessage({
  scheduledMessageId: '<SCHEDULED_MESSAGE_ID>',
});

更多的資訊，請參考 Slack 官方文件 chat.deleteScheduledMessage。
