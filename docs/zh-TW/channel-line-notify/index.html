LINE Notify

什麼是 LINE Notify

LINE Notify 是一個 LINE 的服務，讓你可以發送通知給訂閱者。 一但用戶訂閱了你在 LINE Notify 上的服務，該用戶就會自動加入 LINE Notify 為好友，而且你透過 LINE Notify 發送訊息不需支付額外費用。

在 Bottender 中如何使用 LINE Notify

如果你想要有一個乾淨的自訂 LINE Nnotify 專案，你可以從這個範例開始開發你的專案。 你可以按照以下四個步驟來建立的你的專案：

  1. 從 line-notify 範例下載程式碼。
  2. 執行yarn 或 npm install 指令來安裝所有套件。
  3. 在 .env 填入正確的值。
  4. 執行 yarn dev 或 npm run dev 來啟動開發 Server。

如果你想要擁有我們推薦的資料夾結構，你可以從 create-bottender-app 指令開始並且依照以下的遷移指令來遷移。

建立 LINE Notify 服務

打開這個連結、填寫並送出表單來建立一個新的 LINE Notify 服務。

callback URL 的值應該看起來像這樣：https://{your domain}.ngrok.io/notify/redirect

官方 API 文件：https://notify-bot.line.me/doc/en/

注意： ngrok 是一個知名的服務，透過 Tunnel 提供對外的 HTTPS URL 給你的本地伺服器。 在你於本地開發你的機器人時，這很方便。 你可能會想要在開發時使用它。

環境變數設定

如果你熟悉任何的官方 Bottender 範例，你可能已經知道要如何使用 .env 檔案來在你的本地專案中管理你的環境變數。

在本範例當中，你需要加入 LINE_NOTIFY_CLIENT_ID、LINE_NOTIFY_CLIENT_SECRET 和 ROOT_PATH 到 .env 檔，所以你的檔案會有這些 LINE 相關的環境變數：

LINE_ACCESS_TOKEN={your LINE access token from LINE Messaging API channel}
LINE_CHANNEL_SECRET={your LINE channel secret from LINE Messaging API channel}

LINE_NOTIFY_CLIENT_ID={your LINE Notify client id}
LINE_NOTIFY_CLIENT_SECRET={your LINE Notify client secret}
ROOT_PATH={the ngrok public link}

準備 LINE Notify 實體

LINE Notify 使用 OAuth2 作為授權機制。 最簡流程如下：

  1. 引導用戶到 LINE Notify 授權頁面。
  2. 從 LINE Notify redirect 收到授權碼。
  3. 用授權碼交換 access token。
  4. 使用 access token 來發送通知。

Bottender 提供了 LineNotify 類別來協助你處理 LINE Notify APIs。 你可以建立一個 LINENotify 實體備用。

在專案根目錄下建立一個 lineNotify.js 檔案並且將以下程式碼複製進去：

const { LineNotify } = require('bottender');

export default new LineNotify({
  clientId: process.env.LINE_NOTIFY_CLIENT_ID,
  clientSecret: process.env.LINE_NOTIFY_CLIENT_SECRET,
  redirectUri: `${process.env.ROOT_PATH}/notify/redirect`,
});

引導用戶到 LINE Notify 授權頁面

要服務網頁，你需要添加這些額外的路由到伺服器。 幸運地，自訂 Server 可以解決這個問題！

你可以使用 express、koa、restify 或任何你喜歡的，不過我們將會在這份指南中使用 express。 再繼續往下之前，請確認你有依照這份指南正確設定。

編輯 src/index.js 來發送授權連結給你的用戶：

const lineNotify = require('../lineNotify');

module.exports = async function App(context) {
  const url = lineNotify.getAuthLink('test');
  await context.sendText(url);
};

從 Redirect 接收授權碼並且使用 Access Token 發送通知

加入以下的程式碼片段到 server.js 來處理來自 LINE Notify 的 redirect。

const lineNotify = require('../lineNotify');
// ...

app.prepare().then(() => {
  //...

  server.get('/notify/redirect', async function (req, res) {
    const code = req.query.code;
    const token = await lineNotify.getToken(code);
    await lineNotify.sendNotify(token, 'Hello bottender!');
    res.send('Subscribe successfully. Please close this page.');
  });

  // routes for webhook request
  // ...
});

你可以在 Connected Services 頁面找到你的所有訂閱。

注意： 為了在之後能發送通知，你應該儲存 access tokens。 為了簡化這個範例，我們在這裡就不儲存了。

限制

  1. 訊息格式只允許文字、圖片和基本貼圖，所以你不能發送例如一則帶有按鈕的訊息。
  2. 你不能發送一則超過 1000 個字元的文字訊息。
  3. 每個 token 每小時最多能發送 1000 則訊息。
