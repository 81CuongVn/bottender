Messenger Profile

介紹

你的粉絲專頁上的 Messenger Profile 是給你設置以下各種 Messenger 平台功能的屬性的地方。

  • 開始使用按鈕
  • Persistent Menu
  • 問候語
  • 破冰訊息
  • 網域白名單
  • 連結帳號

所有的設定都可以放到 bottender.config.js 的 channels.messenger.profile 欄位中：

// bottender.config.js
module.exports = {
  channels: {
    messenger: {
      enabled: true,
      path: '/webhooks/messenger',
      pageId: process.env.MESSENGER_PAGE_ID,
      accessToken: process.env.MESSENGER_ACCESS_TOKEN,
      appId: process.env.MESSENGER_APP_ID,
      appSecret: process.env.MESSENGER_APP_SECRET,
      verifyToken: process.env.MESSENGER_VERIFY_TOKEN,
      profile: {
        // ...put your messenger profile setting here
      },
    },
  },
};

在正確設定環境變數後(如果還沒，請先看一下這個)，你可能會呼叫 messenger profile set 指令來設置 profile：

npx bottender messenger profile set

注意： Messenger Profile API 最多只能每 10 分鐘呼叫 10 次 API。 每個粉絲專頁嚴格遵守這個頻率限制。 如果超標了，你可以在幾分鐘後重試。

你可以使用 messenger profile get 指令來查看所有 messenger profile。

npx bottender messenger profile get

設置開始使用按鈕

粉絲專頁 Messenger 的歡迎畫面會顯示一個 開始使用按鈕。 當這個按鈕被按下時，Messenger 平台會發送一個 messaging_postbacks 事件到你的 webhook。

這可以在 bottender.config.js 下的 channels.messenger.profile.getStarted 欄位設定。

// bottender.config.js
module.exports = {
  channels: {
    messenger: {
      // Omission here...
      profile: {
        getStarted: {
          payload: 'GET_STARTED',
        },
      },
    },
  },
};

設置 Persistent Menu

設置 persistent menu 可以讓你的機器人幫助用戶探索以及更容易在整個對話中使用你提供的功能。

這可以在 bottender.config.js 下的 channels.messenger.profile.persistentMenu 欄位設定。

// bottender.config.js
module.exports = {
  channels: {
    messenger: {
      // Omission here...
      profile: {
        getStarted: {
          payload: 'GET_STARTED',
        },
        persistentMenu: [
          {
            locale: 'default',
            composerInputDisabled: false,
            callToActions: [
              {
                type: 'postback',
                title: 'Talk to an agent',
                payload: 'CARE_HELP',
              },
              {
                type: 'postback',
                title: 'Outfit suggestions',
                payload: 'CURATION',
              },
              {
                type: 'web_url',
                title: 'Shop now',
                url: 'https://www.originalcoastclothing.com/',
                webviewHeightRatio: 'full',
              },
            ],
          },
        ],
      },
    },
  },
};

注意：如果你使用任何 messengerExtensions 設定為 true 的 web_url 按鈕，你必須要設定 URL 的網域到白名單網域。

禁止用戶輸入

如果想要禁止用戶輸入，你可以將 composerInputDisabled 屬性設定為 true：

{
  locale: 'default',
  composerInputDisabled: true,
  callToActions: [
    // Omission here...
  ],
}

這表示你的機器人只能透過 persistent menu、postbacks、按鈕和 webviews 來互動。

設置問候語

你的機器人的 Messenger profile 的問候語讓你可以指定用戶將會在你的機器人的歡迎畫面看到的問候語訊息。 這個歡迎畫面是顯示給第一次和你的機器人互動的人們看的。

這可以在 bottender.config.js 下的 channels.messenger.profile.greeting 欄位設定。

// bottender.config.js
module.exports = {
  channels: {
    messenger: {
      // Omission here...
      profile: {
        greeting: [
          {
            locale: 'default',
            text: 'Hello! Welcome to my bot~ ?',
          },
        ],
      },
    },
  },
};

個人化的問候語

你可以使用用戶名稱來個人化問候語文字。 您可以使用下列的樣板字串：

  • {{user_first_name}}
  • {{user_last_name}}
  • {{user_full_name}}

例如：

[
  {
    locale: 'default',
    text: 'Hello {{user_first_name}}! Welcome to my bot~ ?',
  },
];

設置破冰訊息

破冰訊息提供了一個方法讓用戶可以從一個常見問題列表開始進行商業對話。

這可以在 bottender.config.js 下的 channels.messenger.profile.iceBreakers 欄位設定。

// bottender.config.js
module.exports = {
  channels: {
    messenger: {
      // Omission here...
      profile: {
        iceBreakers: [
          {
            question: '<QUESTION>',
            payload: '<PAYLOAD>',
          },
          {
            question: '<QUESTION>',
            payload: '<PAYLOAD>',
          },
        ],
      },
    },
  },
};

question 將會成為由用戶發送的訊息，被發送到對話中而 payload 將會成為 postback webhook 事件的內容。

注意：有一些 profile 元素像是破冰訊息和開始使用按鈕是不能同時使用的。 所以當兩個都被設置時，一個會覆蓋另一個。 以下是由高到低的優先順序：

  • 破冰訊息
  • 開始使用按鈕
  • 透過粉絲專頁 Inbox UI 設置的客製化問題集

設置網域白名單

你的機器人的 Messenger profile 的白名單網域指定了能夠在 Messenger webview 使用 Messenger Extensions SDK、Checkbox Plugin 和 Customer Chat Plugin 的第三方網域的清單。

這可以在 bottender.config.js 下的 channels.messenger.profile.whitelistedDomains 欄位設定。

// bottender.config.js
module.exports = {
  channels: {
    messenger: {
      // Omission here...
      profile: {
        whitelistedDomains: ['<WHITELISTED_DOMAIN>', '<WHITELISTED_DOMAIN>'],
      },
    },
  },
};

注意：網域必須滿足以下條件才能加入白名單：

  • 使用 HTTPS
  • 使用標準網域名稱，例如 https://www.messenger.com/。 白名單是不支援 IP 位址和 localhost 的。
