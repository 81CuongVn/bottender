Rich Menu

Rich Menu 是什麼

Rich Menu 是一個可客製化的選單，它被顯示在聊天視窗上以幫助使用者跟你的 LINE 官方帳號互動。 更多的資訊，你可以參考 官方文件。

要設定幫你的使用者設定 Rich Menu 有四個步驟：

  1. 準備一張 Rich Menu 的圖片
  2. 建立 Rich Menu
  3. 上傳 Rich Menu 的圖片
  4. 連結 Rich Menu 到使用者上

準備一張 Rich Menu 的圖片

Rich Menu 的圖是一個圖片檔案。 你可以在圖片上定義最多 20 個不同的可點擊區塊。 這是一個 Rich Menu 圖片範例：

在這個圖片中，有六個簡單的按鈕。 我們將會在下一個步驟定義這些按鈕的可點擊區塊。

此外，要記住你的 Rich Menu 圖片必須遵守以下的需求：

  • 圖片格式：JPEG 或 PNG
  • 圖片尺寸（像素）：2500x1686、2500x843、1200x810、1200x405,、800x540、800x270
  • 檔案大小上限：1 MB

建立 Rich Menu

在你建立 Rich Menu 之前，你必須建立一個 Rich Menu 物件，用它定義選單的尺寸、可點擊區塊等等。

接著，你可以透過發送一個 HTTP POST 請求到https://api.line.me/v2/bot/richmenu 端點來建立一個 Rich Menu。 你必須把你的 Rich Menu 物件放在 Body 並把你的 Channel Access Token 放進 Authorization Header。

這是一個請求的範例：

  -H 'Content-Type:application/json' \
  -d \
  '{
    "size":{
        "width":2500,
        "height":1686
    },
    "selected":false,
    "name":"Controller",
    "chatBarText":"Controller",
    "areas":[

如果你成功地建立了 Rich Menu，你會拿到一個有 Rich Menu ID 的回應。

上傳 Rich Menu 的圖片

接下來，你可以上傳你在步驟一準備好的 Rich Menu 圖片。 一樣的，你透過發送 HTTP POST 請求到 https://api-data.line.me/v2/bot/richmenu/{richMenuId}/content 端點來上傳圖片。 指定你剛拿到的 Rich Menu ID 作為 richMenuId 路徑參數。

這是請求的範例：

curl -v -X POST https://api-data.line.me/v2/bot/richmenu/{richMenuId}/content \
-H "Authorization: Bearer {channel access token}" \
-H "Content-Type: image/jpeg" \
-T rich_menu.jpg

連結 Rich Menu 到使用者上

在前面的三個步驟，你已經完成了 Rich Menu 的所有建置。 現在，你可以把選單連結到使用者上。 你可以設定 Rich Menu 成為所有使用者的預設 Rich Menu，亦或是連結 Rich Menu 到個別使用者上。

設定預設 Rich Menu

你可以發送一個 HTTP Post 請求到 https://api.line.me/v2/bot/user/all/richmenu/{richMenuId} 端點來設定預設 Rich Menu。

這是請求的範例：

curl -v -X POST https://api.line.me/v2/bot/user/all/richmenu/{richMenuId} \
-H "Authorization: Bearer {channel access token}"

連結 Rich Menu 到個別使用者上

要連結 Rich Menu 到個別使用者上，你可以使用 Bottender 在處理使用者的事件時進行：

await context.linkRichMenu('rich-menu-id');

And to unlink a rich menu from a user, you can use:

await context.unlinkRichMenu();

你依然可以透過發送 HTTP POST 請求到 https://api.line.me/v2/bot/user/{userId}/richmenu/{richMenuId} 和 https://api.line.me/v2/bot/richmenu/bulk/unlink 端點來連結和解除連結 Rich Menu。

這是請求的範例：

連結

curl -v -X POST https://api.line.me/v2/bot/user/{userId}/richmenu/{richMenuId} \
-H "Authorization: Bearer {channel access token}"

解除連結

curl -v -X POST https://api.line.me/v2/bot/richmenu/bulk/unlink \
-H "Authorization: Bearer {channel access token}" \
-H "Content-Type: application/json" \
-d '{
  "userIds":["{userId1}","{userId2}"]
}'

在你連結 Rich Menu 到使用者後，你可以在之後用 Bottender 取得那個 Rich Menu ID：

async function App(context) {
  const richMenu = await context.getLinkedRichMenu();
  console.log(richMenu);
  // {
  //   richMenuId: "rich-menu-id"
  // }
}

你可以在這裡查看完整的範例。

在主選單跟子選單之間切換

現在我們建置了三個 Rich Menu 並且得到了它們的 Rich Menu ID。 我們首先把這些選單 ID 放進我們 Bottender 專案裡的 .env 檔案中。

.env

LINE_ACCESS_TOKEN=
LINE_CHANNEL_SECRET=
MAIN_RICH_MENU_ID=
SUB_RICH_MENU_A_ID=
SUB_RICH_MENU_B_ID=

接著，我們可以檢查 Rich Menu 上的哪個按鈕已經被使用者點擊了並使用 context.linkRichMenu(RICH_MENU_ID) 來在主選單跟它的子選單之間切換。

module.exports = async function App(context) {
  if (context.event.text === 'A') {
    await context.linkRichMenu(process.env.SUB_RICH_MENU_A_ID);
  } else if (context.event.text === 'B') {
    await context.linkRichMenu(process.env.SUB_RICH_MENU_B_ID);
  } else if (context.event.text === 'Back') {
    await context.linkRichMenu(process.env.MAIN_RICH_MENU_ID);
  } else {
    await context.sendText(`User have clicked ${context.event.text}`);
  }
};

如此一來，我們成功地建立了一個 Rich Menu 以及多個子選單。 你可以使用同樣的方法設定更複雜的子選單。
