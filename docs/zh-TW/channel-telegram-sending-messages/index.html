Sending Telegram Messages

Telegram 機器人的基本要素：可更新的訊息和 Reply Markup

簡而言之，你需要知道關於 Telegram 機器人的兩個基本概念：訊息和 Relpy Markup 兩者都是可被更新的。

Telegram 機器人支援許多豐富的訊息格式。 比方說，你可以使用 Markdown 或 HTML 格式來發送 文字訊息。如果你的機器人有付款功能的話你也可以發送發票訊息。 不要錯過了可愛的貼圖訊息。Telegram 幫你託管貼圖集。

Reply Markup 像是在 LINE 上的快速回覆和按紐訊息的組合。 它可以被附加在訊息下面並且提供多個選項來指引你的用戶去開啟一個網頁、觸發機器人的行為或代表用戶回覆一則訊息。

每個訊息和 Reply Markup 都是可被更新的。 如果你傾向在之後更新訊息，記得去管理你的機器人發送過的訊息。

在接下來的程式碼，你可以看到訊息更新的快速範例。

const response = await context.sendMessage('hello');
await context.editMessageText(response.messageId, '*world*', {
  parseMode: 'markdown',
});

Bottender Playground

如果你等不及要嘗試 Telegram 機器人的功能，我們準備了一個簡單的展示機器人給你，Bottender Playground。

在接下來的章節，你可以看到在 Bottender 中使用 Telegram APIs 的簡短介紹。

發送文字訊息

使用這個方法來發送文字訊息。 你可以選擇要以純文字、Markdown 或 HTML 格式來發送訊息。

當發送成功時，發送的訊息會被返回。 你可以儲存返回的訊息以進行進一步的更新。

await context.sendMessage('hello');

更多資訊，請參考 Telegram 的官方文件 sendMessage。

Markdown 風格的 Parse Mode

使用這個方法以 Markdown 來發送文字訊息。

當發送成功時，發送的訊息會被返回。 你可以儲存返回的訊息以進行進一步的更新。

await context.sendMessage('*hello*', { parseMode: 'markdown' });

你可以看到以下是可用的 Markdown 格式。

更多資訊，請參考 Telegram 的官方文件 Formatting Options。

HTML 風格的 Parse Mode

使用這個方法以 HTML 來發送文字訊息。

當發送成功時，發送的訊息會被返回。 你可以儲存返回的訊息以進行進一步的更新。

<b>bold</b>, <strong>bold</strong> <i>italic</i>, <em>italic</em>
<a href="http://www.example.com/">inline URL</a>
<a href="tg://user?id=123456789">inline mention of a user</a>
<code>inline fixed-width code</code>
<pre>pre-formatted fixed-width code block</pre>

await context.sendMessage('<b>hello</b>', { parseMode: 'html' });

更多資訊，請參考 Telegram 的官方文件 Formatting Options。

發送 Rich Media 訊息

在 Telegram 發送 Rich Media 是直接和一致的，在大多數的情況下，你只需要準備網址。

圖片

使用這個方法來發送圖片訊息。 當發送成功時，發送的訊息會被返回。

await context.sendPhoto('https://www.example.com/example.png');

更多資訊，請參考 Telegram 的官方文件 sendPhoto。

音訊

使用這個方法來發送音訊訊息。 當發送成功時，發送的訊息會被返回。

await context.sendAudio('https://www.example.com/example.mp3');

更多資訊，請參考 Telegram 的官方文件 sendAudio。

文件

使用這個方法來發送一般檔案訊息。 當發送成功時，發送的訊息會被返回。 機器人可以發送 50 MB 以下任何類型的檔案，這個限制可能會在未來修改。

await context.sendDocument('https://www.example.com/example.gif');

更多資訊，請參考 Telegram 的官方文件 sendDocument。

影片

使用這個方法來發送影片檔案；Telegram 支援 mp4 影片格式 (其它格式請使用 Document 來發送)。 當發送成功時，發送的訊息會被返回。 機器人可以發送 50 MB 以下的影片檔，這個限制可能會在未來修改。

await context.sendVideo('https://www.example.com/example.mp4');

更多資訊，請參考 Telegram 的官方文件 sendVideo。

動畫

使用這些方法來發送動畫檔案(GIF or H.264/MPEG-4 AVC 影像不包含音訊)。 當發送成功時，發送的訊息會被返回。 機器人可以發送 50 MB 以下的動畫檔，這個限制可能會在未來修改。

await context.sendAnimation('https://www.example.com/example.mp4');

更多資訊，請參考 Telegram 的官方文件 sendAnimation。

貼圖

使用這個方法來發送靜態 .WEBP 或動態 .TGS 貼圖。 當發送成功時，發送的訊息會被返回。

await context.sendSticker('CAADAgADQAADyIsGAAE7MpzFPFQX5QI');

如果你想知道現有貼圖的貼圖ID，你可以參考以下的程式碼。

module.exports = async function App(context) {
  if (context.event.isSticker) {
    await context.sendText(
      `received the sticker: ${context.event.sticker.fileId}`
    );
  }
};

更多資訊，請參考 Telegram 的官方文件 sendSticker。

語音

使用這個方法來發送語音檔，如果你想讓 Telegram 客戶端展示可播放的語音訊息。

你的聲音檔必須是以 OPUS 編碼的 .ogg 檔(其它格式可以用音訊或文件發送) 當發送成功時，發送的訊息會被返回。 機器人可以發送 50 MB 以下的語音訊息，這個限制可能會在未來修改。

await context.sendVoice('https://www.example.com/example.ogg');

更多資訊，請參考 Telegram 的官方文件 sendVoice。

VideoNote

在 Telegram 4.0 後支援長達一分鐘的圓角矩形的 mp4 影片。 使用這個方法來發送影片訊息。 當發送成功時，發送的訊息會被返回。

await context.sendVideoNote('https://www.example.com/example.mp4');

更多資訊，請參考 Telegram 的官方文件 sendVideoNote。

MediaGroup

使用這個方法將一群圖片或影片作為相簿發送。 當發送成功時，發送的訊息會被返回。

await context.sendMediaGroup([
  { type: 'photo', media: 'https://http.cat/100' },
  { type: 'photo', media: 'https://http.cat/101' },
]);

更多資訊，請參考 Telegram 的官方文件 sendMediaGroup。

地點

使用這個方法來發送地圖上的一個地點 當發送成功時，發送的訊息會被返回。

await context.sendLocation({ latitude: 25.105497, longitude: 121.597366 });

更多資訊，請參考 Telegram 的官方文件 sendLocation。

Venue

使用這個方法來發送一個 Venue 。 當發送成功時，發送的訊息會被返回。

await context.sendVenue({
  location: { latitude: 25.105497, longitude: 121.597366 },
  title: 'taipei',
  address: 'taipei address',
});

更多資訊，請參考 Telegram 的官方文件 sendVenue。

Contact

使用這個方法來發送聯絡資訊。 當發送成功時，發送的訊息會被返回。

await context.sendContact({
  phoneNumber: '123456',
  firstName: 'first',
});

如果你有可選參數，將它放在第二個引數。

await context.sendContact(
  {
    phoneNumber: '123456',
    firstName: 'first',
  },
  {
    lastName: '213',
  }
);

更多資訊，請參考 Telegram 的官方文件 sendContact。

投票

使用這個方法來發送投票訊息。 投票訊息不能被發送到私人聊天當中。 當發送成功時，發送的訊息會被返回。

await context.sendPoll('Which one is your favorite food?', [
  '🍔',
  '🍕',
  '🌮',
  '🍱',
]);

更多資訊，請參考 Telegram 的官方文件 sendPoll。

注意：如果你嘗試發送投票訊息到私人聊天當中，你會看到像這樣的錯誤訊息： "description": "Bad Request: polls can't be sent to private chats"。

發票

在你發送發票訊息之前，你必須為你的 Telegram 機器人啟用付款提供者。 使用這個方法來發送發票訊息。 當發送成功時，發送的訊息會被返回。

const invoice = {
  title: 'product name',
  description: 'product description',
  payload: 'bot-defined invoice payload',
  providerToken: 'PROVIDER_TOKEN',
  startParameter: 'pay',
  currency: 'USD',
  prices: [
    { label: 'product', amount: 11000 },
    { label: 'tax', amount: 11000 },
  ],
};
await context.sendInvoice(invoice);

更多資訊，請參考 Telegram 的官方文件 sendInvoice。

ChatAction

使用這個方法讓你的機器人更像真人。 這個狀態會被設定在五秒以內(當你的機器人發送的訊息送達時，Telegram client 會清除他的打字狀態)。 成功時，返回 True。

await context.sendChatAction('typing');

文字訊息使用 typing， 圖片訊息使用 upload_photo， 影片訊息使用 record_video 或 upload_video， 音訊檔案使用 record_audio or upload_audio， 一般檔案使用 upload_document， 地點資訊使用 find_location， Video note 使用 record_video_note 或 upload_video_note。

更多資訊，請參考 Telegram 的官方文件 sendChatAction。

轉傳訊息

使用這個方法來轉傳任何類型的訊息 當發送成功時，發送的訊息會被返回。

const chatId = 313534466;
await context.forwardMessageFrom(chatId, 'messageId', {
  disableNotification: true,
});

const chatId = 413534466;
await context.forwardMessageTo(chatId, 'messageId', {
  disableNotification: true,
});

更多資訊，請參考 Telegram 的官方文件 forwardMesasge。

發送 Reply Markup

Reply Markup 提供了有效的引導機器人用戶進行下一個可能的行動。 Telegram 當中的 Reply Markup 像是在 LINE 上的快速回覆和按紐訊息的組合。

Inline Keyboard

這個物件代表一個 inline keyboard，顯示在它所屬的訊息旁邊。

const replyMarkup = {
  inlineKeyboard: [
    [
      {
        text: 'hi',
        url: 'https://www.example.com',
      },
      {
        text: 'yo',
        callbackData: 'yo',
      },
    ],
  ],
};

更多資訊，請參考 Telegram 的官方文件，Inline Keyboards Markup、New Inline Keyboards 和 Inline Keyboards(On the Fly)。

Keyboard

每當你的機器人發送一則訊息，它都可以帶著一個有預先指定回覆選項的特殊鍵盤。

Telegram 應用程式收到訊息並且展示你的鍵盤給指定用戶。 點選任何一個按鈕就會立即發送相應的指令。

如果你想要你的鍵盤在點擊後消失，試著設定 oneTimeKeyboard 為 true。 如果你想要提供鍵盤給你提及的用戶，將 selective 設定為 true。

const replyMarkup = {
  keyboard: [
    [
      {
        text: 'hi',
      },
      {
        text: 'yo',
      },
    ],
  ],
};

更多資訊，請參考 Telegram 的官方文件，Keyboards、Reply Keyboards 和 Remove Reply Keyboard。

如何發送 Reply Markup

在下面的程式碼當中，你將會看到如何發送 Reply Markup。

sendText

await context.sendText(text, { replyMarkup });

sendPhoto

await context.sendPhoto(url, { replyMarkup });

sendAudio

await context.sendAudio(url, { replyMarkup });

sendDocument

await context.sendDocument(url, { replyMarkup });

sendVideo

await context.sendVideo(url, { replyMarkup });

sendAnimation

await context.sendAnimation(url, { replyMarkup });

sendSticker

await context.sendSticker(sticker, { replyMarkup });

sendVoice

await context.sendVoice(url, { replyMarkup });

sendVideoNote

await context.sendVideoNote(url, { replyMarkup });

sendMediaGroup

await context.sendMediaGroup([media], { replyMarkup });

sendLocation

await context.sendLocation(location, { replyMarkup });

sendVenue

await context.sendVenue(venue, { replyMarkup });

sendContact

await context.sendContact(contact, { replyMarkup });

更新訊息

Telegram 提供機器人更新訊息的能力。 Telegram 機器人甚至可以刪除自己發送過的訊息。

接下來你會看到更新訊息的範例程式。

更多資訊，請參考 Telegram 的官方文件 Update Messages。

更新文字

const response = await context.sendMessage('hello');
await context.editMessageText(response.messageId, '*world*', {
  parseMode: 'markdown',
});

更多資訊，請參考 Telegram 的官方文件 Edit Message Text。

更新標題

const response = await context.sendPhoto('https://http.cat/302', {
  caption: `original caption`,
});
await context.editMessageCaption(response.messageId, 'new caption');

更多資訊，請參考 Telegram 的官方文件 Edit Message Caption。

更新媒體

const response = await context.sendPhoto('https://http.cat/100');
await context.editMessageMedia(response.messageId, {
  type: 'photo',
  media: 'https://http.cat/302',
});

更多資訊，請參考 Telegram 的官方文件 Edit Message Media。

更新 ReplyMarkup

const replyMarkup = {
  inlineKeyboard: [
    [
      {
        text: 'hi',
        url: 'https://www.example.com',
      },
      {
        text: 'yo',
        callbackData: 'yo',
      },
    ],
  ],
};
const response = await context.sendMessage('hello');
await context.editMessageReplyMarkup(response.messageId, replyMarkup);

更多資訊，請參考 Telegram 的官方文件 Edit Message Reply Markup。

刪除訊息

const response = await context.sendMessage('hello');
await context.deleteMessage(response.messageId);

更多資訊，請參考 Telegram 的官方文件 Delete Message。

頻率限制

目前 Telegram 還不支援批次發送訊息，例如推播。

為了避免觸及頻率限制，例如每秒發送訊息給 30 位用戶，可以考慮把訊息散布到更長的時間例如 8-12 小時內發送。 否則，你會得到 429 錯誤。

更多資訊，請參考 Telegram 的官方文件 Rate Limit。
