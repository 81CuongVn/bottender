Multi-Channel Support

Bottender 打算要符合企業專案的需求。 因此，Bottender 在非常早期的階段就支援多個聊天平台。 例如，你可以把每個聊天平台的環境變數放在一個檔案；你可以使用平台特有的路由來從各種平台中組織使用者的事件。

此外，Bottender 的目標是要支援每一個聊天平台的所有功能，例如：Slack 的 Block Kit、LINE 的 Rich Menu、Messenger 的 Handover Protocol 。 而且你不需要學習額外的通用聊天 UI 並擔心他們是否可以套用到每個聊天平台的最新聊天 UI。

打造一個跨平台機器人的主要三個步驟如下：

  1. 設定每個聊天平台
  2. 啟用每個聊天平台
  3. 開發平台的機器人 Action

設定每個聊天平台

要打造一個跨平台機器人，首先你必須設定每一個平台。 簡而言之，你必須為你的機器人所支援的聊天平台在你的 .env 填入環境變數。

# .env

MESSENGER_PAGE_ID=
MESSENGER_ACCESS_TOKEN=
MESSENGER_APP_ID=
MESSENGER_APP_SECRET=
MESSENGER_VERIFY_TOKEN=

WHATSAPP_ACCOUNT_SID=
WHATSAPP_AUTH_TOKEN=
WHATSAPP_PHONE_NUMBER=

LINE_ACCESS_TOKEN=
LINE_CHANNEL_SECRET=

TELEGRAM_ACCESS_TOKEN=

SLACK_ACCESS_TOKEN=
SLACK_VERIFICATION_TOKEN=

VIBER_ACCESS_TOKEN=

注意：如果你不熟悉設定聊天平台，你可以參考以下的 Bottender 文件：

  • Messenger Setup
  • WhatsApp Setup
  • LINE Setup
  • Slack Setup
  • Telegram Setup
  • Viber Setup

啟用每個聊天平台

接下來，你必須確保你已經在 bottender.config.js 啟用了你想要支援的平台。

例如，如果你想要在你的機器人啟用 Messenger，你必須把channels.messenger.enabled 設為 true。

// bottender.config.js

module.exports = {
  channels: {
    messenger: {
      enabled: true,
      path: '/webhooks/messenger',
      pageId: process.env.MESSENGER_PAGE_ID,
      accessToken: process.env.MESSENGER_ACCESS_TOKEN,
      appId: process.env.MESSENGER_APP_ID,
      appSecret: process.env.MESSENGER_APP_SECRET,
      verifyToken: process.env.MESSENGER_VERIFY_TOKEN,
    },
    whatsapp: {
      enabled: true,
      path: '/webhooks/whatsapp',
      accountSid: process.env.WHATSAPP_ACCOUNT_SID,
      authToken: process.env.WHATSAPP_AUTH_TOKEN,
      phoneNumber: process.env.WHATSAPP_PHONE_NUMBER,
    },
    line: {
      enabled: true,
      path: '/webhooks/line',
      accessToken: process.env.LINE_ACCESS_TOKEN,
      channelSecret: process.env.LINE_CHANNEL_SECRET,
    },
    telegram: {
      enabled: true,
      path: '/webhooks/telegram',
      accessToken: process.env.TELEGRAM_ACCESS_TOKEN,
    },
    slack: {
      enabled: true,
      path: '/webhooks/slack',
      accessToken: process.env.SLACK_ACCESS_TOKEN,
      signingSecret: process.env.SLACK_SIGNING_SECRET,
    },
    viber: {
      enabled: true,
      path: '/webhooks/viber',
      accessToken: process.env.VIBER_ACCESS_TOKEN,
      sender: {
        name: 'xxxx',
      },
    },
  },
};

開發平台的機器人 Action

現在，你可以開發 Action 來處理從不同平台來的訊息。

一個正確的機器人 Action 與傳入事件的當前用戶狀態有關。 在下面的章節，你可以看到有多種不同的方法可以來處理跨平台的事件。

使用 context.platform 來管理跨平台事件

你可以使用 context.platform 來檢查每個訊息是屬於哪個平台。

在以下的程式碼中，你可以看到一個簡單的 Action 用來回覆使用者他們正在使用的平台。

module.exports = async function App(context) {
  await context.sendText(`Hello World. Platform: ${context.platform}`);
};

你可以在這裡查看完整的範例。

使用 router 和 platform 來管理跨平台事件

不同的平台為了更好的使用者體驗而提供了不同的進階事件。 例如，Messenger 支援 Button Template，而 LINE 支援 Flex 訊息。 因此，你可能會想要定義一些平台特有的 Action 來處理這些進階事件。

要把機器人的 Action 組織的更好，你可以使用從 bottender/router 來的 router 和 platform 函式來處理不同平台之間的路由。

在這個範例應用程式中，你可以看到兩個額外的 Action 來利用平台特有的事件。

import { platform, router } from 'bottender/router';

async function LineAction(context) {
  await context.sendFlex('This is a line-specific message!', {
    type: 'bubble',
    body: {
      type: 'box',
      layout: 'horizontal',
      contents: [
        {
          type: 'text',
          text: 'Hello,',
        },
        {
          type: 'text',
          text: 'World',
        },
      ],
    },
  });
}

async function MessengerAction(context) {
  await context.sendButtonTemplate('This is a messenger-specific message!', [
    {
      type: 'postback',
      title: 'Hello',
      payload: 'Hello',
    },
    {
      type: 'postback',
      title: 'World',
      payload: 'World',
    },
  ]);
}

module.exports = async function App(context) {
  return router([
    platform('line', LineAction),
    platform('messenger', MessengerAction),
  ]);
};

使用平台特有的路由來管理跨平台事件

透過 Bottender 1.1+，你可以使用平台特有的路由作為一種方式來組織從各種聊天平台來的事件。 想要學習更多這些特有路由的細節，請參考它們的文件：
