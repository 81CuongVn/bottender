Sending LINE Messages

Reply API 和 Push API

基於成本考量，在發送任何訊息之前，你應該要清楚地了解 Reply API 和 Push API 之間的不同：

  • 使用 Reply API 是免費的。 然而，你的機器人只能在使用者跟你的 LINE 官方帳號互動後回覆它一次。 每次的機器人回覆，你可以發送最多 5 個 訊息物件 (參考 LINE 的官方文件「發送回覆訊息」)。 如果你嘗試回覆超過 5 個訊息物件，你會看到錯誤出現在你的 console。

  • Push API 允許開發者在任何時間發送訊息給使用者。 然而，這可能會根據你的國家的價格方案產生一些費用。

預設情況下，Bottender 使用 Reply API 來在對話中發送訊息給使用者。

注意：Reply API 只能在使用者跟你的 LINE 官方帳號互動後的 30 秒內運作。

批次處理訊息

預設情況下，Bottender 在 Batch 模式中執行。 要回覆的訊息會被放到佇列中，並用一個請求送出去給 LINE。

要用兩個訊息發送「hello」和「world」，你可以使用下面其中一種用法：

  1. 呼叫 context.sendText() 兩次：
await context.sendText('hello');
await context.sendText('world');
  1. 用兩個訊息物件來呼叫 send()：
await context.send([
  {
    type: 'text',
    text: 'hello',
  },
  {
    type: 'text',
    text: 'world',
  },
]);

上面的兩個範例是相等的。 然而，我們推薦使用第一種用法因為它對其他平台提供了更好的相容性。

發送文字訊息

文字訊息是在所有通訊平台中最常見以及最常使用的訊息類型 。 當你想攜帶一些動態資料時，它提供了最小型的格式，例如：股價和天氣資訊。

純文字

以下的範例展現了要如何用純文字來回覆。

async function SendHi(context) {
  await context.sendText('Hi!');
}

更多的資訊，請參考 LINE 官方文件文字訊息。

包含 LINE emoji 的文字

你可以在文字訊息使用對應的碼加上 LINE 的原創 Emoji（通常包含 LINE Friends）。 你可以在 Emoji 清單 找到 LINE Emoji 的清單。

async function SendHi(context) {
  await context.sendText(`${String.fromCodePoint(0x100084)} Hi!`);
}

發送 Rich Media 訊息

LINE 的 Rich Media 訊息 包含貼圖、圖片、影片、音訊、地點以及 imagemap。 當你的目標放在抓住使用者的注意力時，例如：限量的促銷，Rich Media 訊息非常有用。 此外，在要建立一個沈浸式的體驗時，例如：講故事，也很方便。

貼圖

透過貼圖，LINE 創造了一種泛用的、 富表達的語言。 貼圖讓你的機器人富有表現力和吸引力。 要發送貼圖，你需要指定貼圖的 Package ID 和 Sticker ID。

await context.sendSticker({
  packageId: '1',
  stickerId: '1',
});

更多的資訊，請參考 LINE 官方文件貼圖訊息。

注意：你只能發送 LINE 原生的 貼圖。 你可以在貼圖清單中找到貼圖的 Package ID 和 Sticker ID。

圖片

要發送 圖片，你需要準備原始圖片的 URL以及比較小的預覽圖。 使用者可以在聊天中看到預覽圖。 當使用者點擊預覽圖，他可以看到原始的圖片。

如果你想要在圖片上面設定一個 Call To Action，你可以參考 Imagemap。

await context.sendImage({
  originalContentUrl: 'https://example.com/image.jpg',
  previewImageUrl: 'https://example.com/preview.jpg',
});

更多的資訊，請參考 LINE 官方文件 圖片訊息。

注意： URL 必須使用 TLS 1.2 或以上的 HTTPS。

影片

要發送一個 影片，你需要準備影片檔案的 URL 和預覽圖的 URL。 這個使用者可以在點擊預覽圖時播放影片。

await context.sendVideo({
  originalContentUrl: 'https://example.com/video.mp4',
  previewImageUrl: 'https://example.com/preview.jpg',
});

更多的資訊，請參考 LINE 官方文件影片訊息。

注意： URL 必須使用 TLS 1.2 或以上的 HTTPS。

音訊

要發送 音訊 檔，你需要準備檔案的 URL 以及音訊檔的時間長度。

await context.sendAudio({
  originalContentUrl: 'https://example.com/audio.mp3',
  duration: 240000,
});

更多的資訊，請參考 LINE 官方文件音訊訊息。

注意： URL 必須使用 TLS 1.2 或以上的 HTTPS。

地點

要把你的地點資訊傳送給使用者，你必須準備標題、地址和經緯度。

await context.sendLocation({
  title: 'my location',
  address: '〒150-0002 東京都渋谷区渋谷２丁目２１−１',
  latitude: 35.65910807942215,
  longitude: 139.70372892916203,
});

更多的資訊，請參考 LINE 官方文件地點訊息。

Imagemap

Imagemap 提供非常彈性和互動式的用法。 它是一個圖片包括多個可點擊的區塊。 當使用者點擊區塊中的其中之一，使用者可以連結到一個網頁或是用他的身份送出一個訊息。

const imagemap = {
  baseUrl: 'https://example.com/bot/images/rm001',
  baseSize: {
    height: 1040,
    width: 1040,
  },
  actions: [
    {
      type: 'uri',
      linkUri: 'https://example.com/',
      area: {
        x: 0,
        y: 0,
        width: 520,
        height: 1040,
      },
    },
    {
      type: 'message',
      text: 'hello',
      area: {
        x: 520,
        y: 0,
        width: 520,
        height: 1040,
      },
    },
  ],
};
const altText = 'this is an imagemap';
await context.sendImagemap(altText, imagemap);

更多的資訊，請參考 LINE 官方文件 Imagemap 訊息。

發送 Template 訊息

Template 訊息 是一個由圖片、影片、標題、子標題和按鈕所組成的互動式畫廊。

Template 訊息 是提供 Rich Media 互動的關鍵。 這常被使用在顯示多個選項以及下一步的動作給使用者的情境，例如：套用優惠券、預定房間、預約。

注意： 相較於 Template 訊息，在 Flex 訊息可以使用之後，我們就高度地依賴他。 有兩個主要的原因：

  • Flex 訊息 同時支援桌面跟手機版裝置，而 Template 訊息只支援行動裝置。
  • Flex 訊息 是一種跟 HTML 類似的聊天 UI，可以創造更好、更有吸引力的使用者體驗。
const template = {
  type: 'buttons',
  thumbnailImageUrl: 'https://example.com/bot/images/image.jpg',
  title: 'Menu',
  text: 'Please select',
  actions: [
    {
      type: 'postback',
      label: 'Buy',
      data: 'action=buy&itemid=123',
    },
    {
      type: 'postback',
      label: 'Add to cart',
      data: 'action=add&itemid=123',
    },
    {
      type: 'uri',
      label: 'View detail',
      uri: 'http://example.com/page/123',
    },
  ],
};
const altText = 'this is a template';
await context.sendTemplate(altText, template);

發送 Flex 訊息

發送快速回覆

我們建議你將快速回覆視為使用者輸入文字的一種替代方式。

當使用者從一個 LINE 官方帳號收到一個包含 快速回覆按鈕 的訊息時，這些按鈕出現在聊天視窗的底部。 這個使用者可以點擊其中一個按鈕作為回覆。

一個快速回覆由最多13 個快速回覆按鈕所組成。 當使用點擊一個快速回覆按鈕，該快速回覆會消失。 接著，被點擊的按鈕的標題會被發在對話中作為回應。 同時，這個使用者會觸發快速回覆按鈕所綁上的 Action。

快速回覆可以用在跟 LINE 跟方帳號的 1 對 1 聊天、群組以及房間中。

const quickReply = {
  items: [
    {
      type: 'action',
      action: {
        type: 'cameraRoll',
        label: 'Send photo',
      },
    },
    {
      type: 'action',
      action: {
        type: 'camera',
        label: 'Open camera',
      },
    },
  ],
};

更多的資訊，請參考 LINE 官方文件 使用快速回覆。

發送快速回覆

因為 快速回覆 是用來代替使用者輸入，你必須送出 快速回覆 作為其他訊息的附件，例如：文字訊息， Template 訊息 或 Flex。

針對訊息發送快速回覆

await context.send([
  {
    type: 'text',
    text: 'hello',
    quickReply,
  },
]);

針對文字訊息發送快速回覆

await context.sendText('hello', { quickReply });

針對 Template 訊息發送快速回覆

await context.sendTemplate(altText, template, { quickReply });

針對 Buttons Template 訊息發送快速回覆

await context.sendButtonTemplate(altText, template, { quickReply });

針對 Confirm Template 訊息發送快速回覆

await context.sendConfirmTemplate(altText, template, { quickReply });

針對 Carousel Template 訊息發送快速回覆

await context.sendCarouselTemplate(altText, template, { quickReply });

針對 Flex 訊息發送快速回覆

await context.sendFlex(altText, contents, { quickReply });

在接下來的章節，你可以看到多種不同類型的快速回覆。

文字快速回覆

當這個 Action 被點擊，在 text 屬性中的字串會被作為從使用者來的訊息送出。

const quickReply = {
  items: [
    {
      type: 'action',
      action: {
        type: 'message',
        label: 'Yes',
        text: 'Yes',
      },
    },
  ],
};

更多資訊，請參考 LINE 的官方文件 Message Actions。

Postback 快速回覆

當這個 Action 被點擊，一個 Postback 事件會藉由 Webhook 回來，帶著在 data 屬性的特定字串。

const quickReply = {
  items: [
    {
      type: 'action',
      action: {
        type: 'postback',
        label: 'Buy',
        data: 'action=buy&itemid=111',
        text: 'Buy',
      },
    },
  ],
};

更多資訊，請參考 LINE 的官方文件 Postback Actions。

URI 快速回覆

當 Action 被執行時會打開 URI。 可以使用的 URI Scheme 有 http、https、line 以及 tel。 針對 LINE URL Scheme，你可以觸發以下的 Action：

  • 打開相機跟相簿
  • 打開地點畫面
  • 分享你的 LINE 官方帳號
  • 打開你的 LINE 官方帳號的 Timeline 跟帳號頁面
  • 發送文字訊息
  • 打開 Profile 資訊
  • 打開常用 LINE 畫面
  • 打開 LINE 設定畫面
  • 打開貼圖商店
  • 打開主題商店
  • 用 LINE Out 打電話
  • 打開一個 LIFF 應用程式
const quickReply = {
  items: [
    {
      type: 'action',
      action: {
        type: 'uri',
        label: 'View details',
        uri: 'http://example.com/page/222',
        altUri: {
          desktop: 'http://example.com/pc/page/222',
        },
      },
    },
  ],
};

更多資訊，請參考 LINE 的官方文件 URI Actions。

注意：

  • 只有當你在 Flex 訊息中設定 URI Action 時才支援 altUri.desktop 屬性。
  • 更多 LINE URL Scheme 相關的資訊，請看使用 LINE URL Scheme。

Datetime Picker 快速回覆

當關聯到這個 Action 的控制器被點擊，一個 Postback 事件會藉由 Webhook 回來，帶著被使用者從日期時間選擇器選擇的日期和時間。 Datetime Picker action 不支援時區。

有三個不同的模式：

  • date：選一個日期
  • time：選一個時間
  • datetime：選一個日期和時間
const quickReply = {
  items: [
    {
      type: 'datetimepicker',
      label: 'Select date',
      data: 'storeId=12345',
      mode: 'datetime',
      initial: '2017-12-25t00:00',
      max: '2018-01-24t23:59',
      min: '2017-12-25t00:00',
    },
  ],
};

更多資訊，請參考 LINE 的官方文件 Datatime Picker Action。

注意：Datetime Picker Action 只在 iOS 的 LINE 7.9.0 和 Android 的 LINE 7.12.0 同等或以上的版本支援。

發送帶著頭像與顯示名稱的訊息

你可以透過在訊息選項中提供 sender 屬性來指定頭像與顯示名稱：

const sender = {
  name: 'Cony',
  iconUrl: 'https://line.me/conyprof',
};

await context.sendText('Hello, I am Cony!!', { sender });
await context.sendImage(
  {
    originalContentUrl: 'https://example.com/image.jpg',
    previewImageUrl: 'https://example.com/preview.jpg',
  },
  { sender }
);

如果你偏好用 LINE 原始的訊息物件來發送訊息，你可以在訊息物件中提供 sender 屬性：

await context.send([
  {
    type: 'text',
    text: 'Hello, I am Cony!!',
    sender: {
      name: 'Cony',
      iconUrl: 'https://line.me/conyprof',
    },
  },
]);

更多資訊，請參考 LINE 的官方文件 改變頭像與顯示名稱。

頻率限制

就像很多通訊平台，LINE 針對每個端點有頻率限制。 如果你長期持續地超出頻率限制送出請求，你的機器人可能會停止回應。 這是因為 LINE 擋掉了進入你機器人的請求。

如果你嘗試要建置一個有巨大瞬間流量的活動機器人，例如：黑色星期五的超級折扣，你必須小心你的頻率限制。

Send API

Send API，你必須遵守以下的頻率限制：

  • 每分鐘 100,000 個請求
  • 每秒 1,700 個請求
