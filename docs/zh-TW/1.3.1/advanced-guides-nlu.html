Natural Language Understanding

善用 NLU 技術

當需要建立機器人來回答開放性問題時，例如客服 FAQ，或者在機器人內的搜尋引擎，你可能會想要使用 NLU 引擎建立機器人來分析用戶輸入。 為了回覆出正確答案，NLU 的主要任務是分析用戶意圖和關鍵字。

訓練有素的 NLU 引擎對於預測用戶真正的意圖或發現用戶輸入已超出目前的知識庫通常有 90% 以上的準確率。 有時候，找到最佳的意圖結構以及訓練 NLU 引擎需要花時間和一些迭代。

在接下來的章節，你會看到如何整合 Bottender 與各種現代的 NLU 服務：

  • QnA Maker
  • Dialogflow
  • LUIS
  • Rasa

使用 QnA Maker 建置

我們選擇先介紹 QnA Maker 是因為他提供了友善的建置流程。 不像其他的 NLU 服務需要一些時間去建置意圖和撰寫訓練語句。 在 QnA Maker，你只需要複製貼上你的 FAQ，然後 NLU 引擎就準備好了。

第一步：QnA Maker 設定

若想建置一個整合了 QnA Maker 的機器人，你必須建立 QnA Maker 知識庫然後遵循官方指南。

在你發布知識庫之後，你會獲得 RESOURCE_NAME, KNOWLEDGE_BASE_ID, and ENDPOINT_KEY (詳細說明可參閱這裡)。 確保你複製它們到 .env 檔案中。

# .env

RESOURCE_NAME=
KNOWLEDGE_BASE_ID=
ENDPOINT_KEY=

第二步：透過 bottender/qna-maker 連結 Bottender 與 QnA Maker

為了使機器人開發愉快，我們做了bottender/qna-maker 套件。 你可以透過 npm 或 yarn 安裝這個套件。

透過 npm：

npm install @bottender/qna-maker

或透過 yarn：

yarn add @bottender/qna-maker

在接下來的範例程式碼，你將會看到 Bottender 優雅地和 QnA Maker 整合在一起。 而你只需要填寫你的環境變數以及門檻分數，然後 Bottender 就能使用來自 QnA Maker 的答案作為回覆。

const { chain } = require('bottender');
const qnaMaker = require('@bottender/qna-maker');

async function Unknown(context) {
  await context.sendText('Sorry, I don’t know what you say.');
}

const QnaMaker = qnaMaker({
  resourceName: process.env.RESOURCE_NAME,
  knowledgeBaseId: process.env.KNOWLEDGE_BASE_ID,
  endpointKey: process.env.ENDPOINT_KEY,
  scoreThreshold: 70,
});

module.exports = async function App() {
  return chain([
    QnaMaker, //
    Unknown,
  ]);
};

完整的範例程式請參考 Bottender 範例，With QnA Maker。

使用 Dialogflow 建置

Google 創造了 Dialogflow。 自從 2019 年 Google 發表了他們的 NLU 預訓練 BERT, 我們對 Google 的 NLU 解決方案充滿信心；至少我們可以認為這可能是最新技術。

第一步：Dialogflow 設定

若想建置一個整合了 Dialogflow 的機器人，你必須遵循 Dialogflow 的文件 Quickstart: Setup 來設定 Dialogflow，並且填入兩個值到 .env 檔：

  • GOOGLE_APPLICATION_CREDENTIALS，這是一個包含了你的 服務帳號 key 的 JSON 檔的檔案路徑。
  • GOOGLE_APPLICATION_PROJECT_ID，這是 GCP 上的專案 ID
# .env

GOOGLE_APPLICATION_CREDENTIALS=
GOOGLE_APPLICATION_PROJECT_ID=

第二步：建立 Dialogflow Agent

接著你可以遵循這份 Dialogflow 文件 Quickstart: Build an Agent 來建置 Dialogflow agent。 在這個範例，你建立 Agent 並且建立一個名為 greeting 的意圖。你可以在 Dialogflow Console 上設定你這個意圖的訓練語句。

在你完成 Agent 的設定之後，你可以呼叫 Dialogflow 的 API 來分析機器人收到的訊息的意圖。

第二步：透過 bottender/dialogflow 連結 Bottender 與 Dialogflow

為了使機器人開發愉快，我們做了bottender/dialogflow 套件。 你可以透過 npm 或 yarn 安裝這個套件。

透過 npm：

npm install @bottender/dialogflow

或透過 yarn：

yarn add @bottender/dialogflow

在接下來的範例程式碼，你將會看到 Bottender 優雅地和 Dialogflow 整合在一起。 而你只需要填寫你的環境變數以及意圖對應到的函數，例如 greeting 意圖對應到 SayHello 函數。

const { chain } = require('bottender');
const dialogflow = require('@bottender/dialogflow');

async function SayHello(context) {
  await context.sendText('Hello!');
}

async function Unknown(context) {
  await context.sendText('Sorry, I don’t know what you say.');
}

const Dialogflow = dialogflow({
  projectId: process.env.GOOGLE_APPLICATION_PROJECT_ID,
  actions: {
    greeting: SayHello,
  },
});

module.exports = async function App() {
  return chain([
    Dialogflow, //
    Unknown,
  ]);
};

注意：

  • Dialogflow 提供 intent name 和 intent display name。 前者的值在建立後就不可更改；後者的值可以隨時更改。 當撰寫 bottender/dialogflow 的意圖和函數對應時，你可以使用任兩者之一來指定意圖。
  • 完整的範例程式請參考 Bottender 範例，With Dialogflow。

使用 LUIS 建置

第一步：LUIS 設定

若想建置一個整合了 LUIS (Language Understanding Intelligent Service) 的機器人，你必須遵循 Official Setup Guide 在 LUIS portal 建立一個新的應用程式並且填寫三個值：LUIS_APP_ID、LUIS_APP_KEY、LUIS_APP_ENDPOINT 到 .env 檔案中。

# .env

LUIS_APP_ID=
LUIS_APP_KEY=
LUIS_APP_ENDPOINT=

第二步：訓練以及發布你的 LUIS 專案

在這個範例，你會建立一個名為 greeting 的意圖。 你可以在 LUIS console 替這個意圖設定你的訓練語句。 然後你需要訓練模型並且發布它。

第二步：透過 bottender/luis 連結 Bottender 與 LUIS

為了使機器人開發愉快，我們做了bottender/luis 套件。 你可以透過 npm 或 yarn 安裝這個套件。

透過 npm：

npm install @bottender/luis

或透過 yarn：

yarn add @bottender/luis

在接下來的範例程式碼，你將會看到 Bottender 優雅地和 LUIS 整合在一起。 而你只需要填寫你的環境變數和門檻分數以及意圖對應到的函數，例如 greeting 意圖對應到 SayHello 函數。

const { chain } = require('bottender');
const luis = require('@bottender/luis');

async function SayHello(context) {
  await context.sendText('Hello!');
}

async function Unknown(context) {
  await context.sendText('Sorry, I don’t know what you say.');
}

const Luis = luis({
  appId: process.env.LUIS_APP_ID,
  appKey: process.env.LUIS_APP_KEY,
  endpoint: 'https://westus.api.cognitive.microsoft.com',
  actions: {
    greeting: SayHello,
  },
  scoreThreshold: 0.7,
});

module.exports = async function App() {
  return chain([
    Luis, //
    Unknown,
  ]);
};

完整的範例程式請參考 Bottender 範例，With LUIS.ai。

使用 Rasa NLU 建置

如果你正在尋找本地端的 NLU 解決方案，你可能會選擇 Rasa NLU。

第一步：Rasa NLU 設定

若想建置一個整合了 Rasa NLU 的機器人，你必須遵循 Official Installation Guide 來安裝 Rasa。 然後你可以執行以下指令來訓練你的 NLU 模型：

rasa train nlu

這個指令會尋找在 data 資料夾下的訓練資料檔以及將模型儲存在 models 資料夾下。 你可以在 Rasa 的文件 Training Data Format 看到更多有關如何生成訓練資料的資訊。

在你準備好你的 NLU 模型後，你可以執行以下指令：

rasa run --enable-api -m models/nlu-your-model-id.tar.gz

這個指令會在本地的 port 5005 運行一個含有你 NLU 模型的 server。 接下來你可以透過呼叫 /model/parse 端點來對你的模型發出預測請求。 你可以在這裡看到關於這個 API 的文件。

第二步：透過 bottender/rasa 連結 Bottender 與 Rasa

為了使機器人開發愉快，我們做了bottender/rasa 套件。 你可以透過 npm 或 yarn 安裝這個套件。

透過 npm：

npm install @bottender/rasa

或透過 yarn：

yarn add @bottender/rasa

在接下來的範例程式碼，你將會看到 Bottender 優雅地和 Rasa 整合在一起。 而你只需要設定原始 URL以及門檻信心指數，然後寫一個意圖與函數的對應，例如 greeting 意圖對應到 SayHello 函數。

const { chain } = require('bottender');
const rasa = require('@bottender/rasa');

async function SayHello(context) {
  await context.sendText('Hello!');
}

async function Unknown(context) {
  await context.sendText('Sorry, I don’t know what you say.');
}

const Rasa = rasa({
  origin: 'http://localhost:5005',
  actions: {
    greeting: SayHello,
  },
  confidenceThreshold: 0.7,
});

module.exports = async function App() {
  return chain([
    Rasa, //
    Unknown,
  ]);
};

完整的範例程式請參考 Bottender 範例，With Rasa。
