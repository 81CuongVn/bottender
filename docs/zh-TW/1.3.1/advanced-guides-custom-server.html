Running Bottender with Custom Servers

概念

bottender dev 和 bottender start 是由 create-bottender-app 生成的 npm 預設腳本。

因此，你可以在 package.json 找到以下內容：

{
  ...
  "scripts": {
    "dev": "bottender dev",
    "start": "bottender start",
    ...
  },
  ...
}

當執行了 bottender dev 或 bottender start 指令的其中之一，它會在後台設置一個預設的 Express Server 並且載入專案根目錄資料夾下的 index.js 作為聊天機器人的入口 Action。

若想將 Bottender 跑在自訂的 HTTP Server 上，你需要準備一個 HTTP Server，委派 Webhook 路由給 Bottender，並且編輯 package.json 設定。

Express

使用自訂 Express Server 建立一個新專案

如果你想要有一個乾淨的自訂 Express 專案，你可以從這個範例開始開發你的專案。 你可以依照以下這四個步驟來建立你的專案：

  1. 從這個範例下載程式碼。
  2. 執行yarn 或 npm install 指令來安裝所有套件。
  3. 在 .env 填入正確的值。
  4. 執行 yarn dev 或 npm run dev 來啟動開發 Server。

If you want to have the folder structure we recommend, you could start with create-bottender-app command and migrate it to the custom express server by following the migration instructions below.

遷移現有專案到自訂 Express Server

假設你已經有一個透過 create-bottender-app 建立的專案，而且你想要用 Express Server 開發一些額外的 APIs。 在這種情況下，你需要寫一個自訂 Express Server，並將所有聊天機器人的 Webhook 請求委派給 Bottender 應用程式。

你可以依照以下指示來達成這些：

  1. 執行 yarn add express body-parser nodemon 或 npm install express body-parser nodemon 來安裝我們需要的套件。
  2. 在專案根目錄下建立一個 server.js 檔案並且將以下程式碼複製進去：
const bodyParser = require('body-parser');
const express = require('express');
const { bottender } = require('bottender');

const app = bottender({
  dev: process.env.NODE_ENV !== 'production',
});

const port = Number(process.env.PORT) || 5000;

// the request handler of the bottender app
const handle = app.getRequestHandler();

app.prepare().then(() => {
  const server = express();

  server.use(
    bodyParser.json({
      verify: (req, _, buf) => {
        req.rawBody = buf.toString();
      },
    })
  );

  // your custom route
  server.get('/api', (req, res) => {
    res.json({ ok: true });
  });

  // route for webhook request
  server.all('*', (req, res) => {
    return handle(req, res);
  });

  server.listen(port, err => {
    if (err) throw err;
    console.log(`> Ready on http://localhost:${port}`);
  });
});
  1. 將 package.json 當中的腳本修改為 nodemon server.js 和 node server.js：
{
  ...
  "scripts": {
    "dev": "nodemon server.js",
    "start": "node server.js",
    ...
  },
  ...
}

這就是你需要做的所有事情，而且你可以讓你的機器人託管在自訂 Express Server 上！

遷移現有專案到自訂 Koa Server

假設你已經有一個透過 create-bottender-app 建立的專案，而且你想要用 Koa Server 開發一些額外的 APIs。 在這種情況下，你需要寫一個自訂 Koa Server，並將所有聊天機器人的 Webhook 請求委派給 Bottender 應用程式。

你可以依照以下指示來達成這些：

  1. 執行 yarn add koa koa-router koa-bodyparser nodemon 或 npm install koa koa-router koa-bodyparser nodemon 來安裝我們需要的套件。
  2. 在專案根目錄下建立一個 server.js 檔案並且將以下程式碼複製進去：
const Koa = require('koa');
const Router = require('koa-router');
const bodyParser = require('koa-bodyparser');
const { bottender } = require('bottender');

const app = bottender({
  dev: process.env.NODE_ENV !== 'production',
});

const port = Number(process.env.PORT) || 5000;

const handle = app.getRequestHandler();

app.prepare().then(() => {
  const server = new Koa();
  const router = new Router();

  router.get('/api', ctx => {
    ctx.response.body = { ok: true };
  });

  router.all('*', async ctx => {
    await handle(ctx.req, ctx.res);
    ctx.respond = false;
  });

  server.use(bodyParser());
  server.use((ctx, next) => {
    ctx.req.body = ctx.request.body;
    ctx.req.rawBody = ctx.request.rawBody;
    return next();
  });
  server.use(router.routes());

  server.listen(port, err => {
    if (err) throw err;
    console.log(`> Ready on http://localhost:${port}`);
  });
});
  1. 將 package.json 當中的腳本修改為 nodemon server.js 和 node server.js：
{
  ...
  "scripts": {
    "dev": "nodemon server.js",
    "start": "node server.js",
    ...
  },
  ...
}

這就是你需要做的所有事情，而且你可以讓你的機器人託管在自訂 Koa Server 上！

app.prepare().then(() => {
