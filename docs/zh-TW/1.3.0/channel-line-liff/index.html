LINE Front-end Framework (LIFF)

LINE Front-end Framework (LIFF)，應用程式裡的 Webview 解決方案

LIFF 有提供三種不同尺寸嵌入式 Webview。 你可以使用這些尺寸來豐富你的使用者體驗。

此外，LIFF給你更多機會去改善使用者跟機器人之間的互動。 其中一個常見的情境就是在 LINE 官方帳號中提交表單。

LIFF 的功能

LIFF 讓你在你的網頁上能夠取用許多不同的 API。 在你開始之前，你可能會想要先從 LINE 官方文件獲得一些基本知識：

  • Getting user profile
  • Sending messages to LINE
  • Opening link in external browser
  • Scanning QR Code
  • Connecting to Bluetooth devices
  • Others features in LIFF playground

LIFF V2

LIFF v2 導入了一些新功能給開發者，例如：掃描 QR Code、讓使用者在桌面版 LINE 應用程式打開 LIFF 頁面。

為了快速地跟上，這是一個最小的實作 - LIFF v2 的 Bottender 範例。

以下的部分是一步一步的教學，有關如何在 Create Bottender App 建立的 Bottender 專案中用 LIFF 發送訊息。

注意： LIFF v1 已經不再建議使用。 如果你正在使用它，你應該升級到 LIFF v2。 然而，你還是可以找到 LIFF v1 的範例。

流程

要讓你的 LIFF 頁面運作，你典型的工作流程應該會看起來像這樣：

  • 在 LINE Developers Console 建立一個 LINE Login Channel
  • 在 LINE Login Channel 添加一個 LINE 應用程式
  • 開發 LIFF 頁面

建立 LINE Login Channel

首先，你需要打開 LINE Developers Console 來建立一個 LINE Login Channel。

更多跟建立 LINE Login Channel 有關的細解，請參考 建立 Provider 跟 Channel。

把 LIFF 添加到 LINE Login Channel

  • 在你剛建立好的 LINE Login Channel 裡點擊「LIFF」標籤。
  • 點擊「Add」按鈕
  • 用這個格式：https://{your domain}.ngrok.io/liff，填入「Endpoint URL」
  • 填入其他的值

更多註冊 LIFF 應用程式相關的細節，請參考 註冊 LIFF 應用程式。

注意： ngrok 是一個知名的服務，透過 Tunnel 提供對外的 HTTPS URL 給你的本地伺服器。 在你於本地開發你的機器人時，這很方便。 你可能會想要在開發時使用它。

環境變數設定

If you are familiar with any official Bottender example, you may already know about how to use the .env file to manage your environment variables in your local project.

在這個情況下，你需要添加另一個 LINE_LIFF_ID 環境到 .env 檔案，如此一來，你的檔案會有只少這三個 LINE 相關的環境變數：

LINE_ACCESS_TOKEN={your LINE access token from LINE Messaging API channel}
LINE_CHANNEL_SECRET={your LINE channel secret from LINE Messaging API channel}
LINE_LIFF_ID={your LIFF id from LINE Login channel}

你可以在 LIFF URL 上找到你的 LIFF ID。 LIFF URL 的格式會看起來像 line://app/{your LIFF ID}。

更多 LIFF ID 相關的細節，請參考 註冊 LIFF 應用程式 (在步驟 4)。

為 LIFF 頁面添加自訂路由到 HTTP 伺服器

要服務這些 LIFF 頁面，你需要添加這些額外的路由到伺服器。 幸運地，自訂 Server 可以解決這個問題！

你可以使用 express、koa、restify 或任何你喜歡的，不過我們將會在這份指南中使用 express。 再繼續往下之前，請確認你有依照 這份指南 正確設定。

在有了一個自訂 Server 之後，你可以添加以下兩個路由到 server.js 來處理從 LIFF 來的請求。

server.get('/send-id', (req, res) => {
  res.json({ id: process.env.LINE_LIFF_ID });
});

server.get('/liff', (req, res) => {
  const filename = path.join(`${__dirname}/liff.html`);
  res.sendFile(filename);
});

初始化 LIFF 頁面

在開始使用任何 LIFF 提供的功能之前，你需要建立一個 liff.html 檔案在專案的根目錄並複製以下的程式碼進去來初始化 LIFF：

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>BOTTENDER LINE LIFF V2 DEMO</title>
  </head>

  <body>
    <script src="https://static.line-scdn.net/liff/edge/2.1/sdk.js"></script>
    <script>
      function initializeLiff(myLiffId) {
        liff
          .init({
            liffId: myLiffId,
          })
          .then(() => {
            alert('LIFF init success!');
          })
          .catch(err => {
            alert(`error: ${JSON.stringify(err)}`);
          });
      }

      document.addEventListener('DOMContentLoaded', () => {
        fetch(`/send-id`)
          .then(reqResponse => reqResponse.json())
          .then(jsonResponse => {
            let myLiffId = jsonResponse.id;
            initializeLiff(myLiffId);
          })
          .catch(err => {
            alert(`error: ${JSON.stringify(err)}`);
          });
      });
    </script>
  </body>
</html>

值得一提的是，liff.init() 和一些其他的 liff 方法回傳 Promise 作為結果，所以你應該謹慎處理這些非同步的程式碼。

從 LIFF 頁面傳送訊息

在初始化 LIFF 頁面後，我們可以呼叫 liff.sendMessages 來命令式地在對話中傳送訊息。 例如：

liff.sendMessages([
  {
    type: 'text',
    text: 'Hello, LIFF!',
  },
]);

最多可以接受五個文字、圖片、影片、音訊、地點、Template (只有 URI Action 可以被當作 Action) 以及 Flex 訊息。

讓我們來添加一個點擊事件監聽器來在點擊時傳送訊息。 你可以用以下的實作取代 liff.html 裡面的 body 標籤內容：

<body>
  <button id="button">send test message</button>
  <script src="https://static.line-scdn.net/liff/edge/2.1/sdk.js"></script>
  <script>
    function initializeLiff(myLiffId) {
      liff
        .init({
          liffId: myLiffId,
        })
        .then(() => {
          setButtonHandler();
        })
        .catch(err => {
          alert(`error: ${JSON.stringify(err)}`);
        });
    }

    function setButtonHandler() {
      let button = document.getElementById('button');
      button.addEventListener('click', () => {
        window.alert('clicked: sendMessages');
        liff
          .sendMessages([
            {
              type: 'text',
              text: 'Hello, LIFF!',
            },
          ])
          .then(() => {
            alert('message sent');
            liff.closeWindow();
          })
          .catch(err => {
            window.alert('Error sending message: ' + err);
          });
      });
    }

    document.addEventListener('DOMContentLoaded', () => {
      fetch(`/send-id`)
        .then(reqResponse => reqResponse.json())
        .then(jsonResponse => {
          let myLiffId = jsonResponse.id;
          initializeLiff(myLiffId);
        })
        .catch(err => {
          alert(`error: ${JSON.stringify(err)}`);
        });
    });
  </script>
</body>

把 LIFF 連結發送給使用者

要測試這個 LIFF 頁面的話，你可以取代在 index.js 裡面的程式碼來把 LIFF URL 發送給使用者：

module.exports = async function App(context) {
  const liffUrl = `https://liff.line.me/${process.env.LINE_LIFF_ID}`;
  await context.sendText(liffUrl);
};

現在，你可以執行 yarn dev 或 npm run dev 來啟動伺服器，你就會有一個有簡單 LIFF 頁面的 LINE 官方帳號了。
