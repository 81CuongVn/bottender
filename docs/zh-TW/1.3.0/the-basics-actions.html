Resolving Actions

Action 是 Bottender 應用程式的最小建置區塊。 一個 Action 描述了你想要依據對話 Context 做出什麼回應。

要定義一個 Action，只要寫一個 JavaScript 的函式或非同步函式：

async function SayHi(context) {
  await context.sendText('Hi!');
}

Action 通常接收 context 作為第一個參數。 在對話 Context 裡面有非常多的東西，你可以使用它們來打造你的 Action。 舉例來說，你可以使用 context.sendText() 來從機器人送文字給使用者。

組合 Action

Action 可以用其他 Action 作為它們的回傳值。 這讓我們可以在任何的階層使用一樣的 Action 抽象概念。

例如，你可以建立一個 App Action，它會在它收到 hi 的時候回覆 Hi!，收到其他則都回覆抱歉：

async function SayHi(context) {
  await context.sendText('Hi!');
}

async function Unknown(context) {
  await context.sendText('Sorry. I do not understand what you say.');
}

async function App(context) {
  if (context.event.text == 'hi') {
    return SayHi;
  }
  return Unknown;
}

注意：用 Create Bottender App 建立的新 Bottender 應用程式會有一個 App Action 在 src/index.js 檔案裡作為進入點。 然而，如果你非常熟悉 Bottender，你可以幫 Action 改名、甚至使用不同的結構來取代它。

傳遞 Props 給 Action

先前，你只看到 Bottender 的 Action 接一個參數 - context：

async function SayHi(context) {
  await context.sendText('Hi!');
}

然而，你可以取用第二個參數 - props 來定義更彈性的 Action：

async function SayHi(context, props) {
  await context.sendText(`Hi, ${props.name}.`);
}

替代直接回傳一個沒有 props 的 Action，你可以使用 withProps 來傳遞一個物件當作 props 給這個 Action：

const { withProps } = require('bottender');

async function App(context) {
  return withProps(SayHi, { name: 'Bob' });
}

Bottender 將會用 { name: 'Bob' } 當作 props 來呼叫 SayHi Action，並送出 Hi, Bob. 文字訊息給使用者作為結果。

如何除錯

Bottender 在內部使用著名的 debug 套件來收集一些有用的資訊，這些資訊可以在你提供對應的 DEBUG 環境變數時顯示出來。 要除錯你的 Action 時，你可以執行你的指令伴隨著 DEBUG=bottender:action，例如：

DEBUG=bottender:action npm start

Note: If you are developing your bots on Windows, you may use cross-env to assign DEBUG environment variable:

cross-env DEBUG=bottender:action npm start

或者你可以把你的 DEBUG 環境設定放在你的 .env 檔案裡：

DEBUG=bottender:action

我們建議你應該總是幫你的 Action 命名，如此一來，才能印出有意義的路徑給你除錯，而不是顯示 Anonymous 作為 Action 名稱。
